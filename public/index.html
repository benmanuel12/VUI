How to use (Please remember to scroll down, the instructions are quite long)

This program takes one or more text files with basic Markdown formatting and a voice command for what word or phrase to search for inside the file or files
It then returns the results of the search

Accepted commands

File Search
  • please do a search for word/phrase
  • do a search for word/phrase
  • do search for word/phrase
  • search for word/phrase
  • please do a look for word/phrase
  • do a look for word/phrase
  • do look for word/phrase
  • look for word/phrase
For the next list of commands, when a file name is prompted, it is in fact entirely ignored by the program
  • please find word/phrase in file name
  • find word/phrase in file name
  • please find word/phrase
  • find word/phrase

Valid file structure and formatting

All files must have the .txt file extension
If you want the files to be indexed when searching multiple files, one and only one line of the file must begin with '#' followed by your title
This does not have to be at the top of the page but that makes the most sense
All headings must be prefixed with '###'
The rest of the file may be written in any way you like as long as you are aware that '##', '####', '#####' and '######' (the markdown symbols for Headings 2, 4, 5 and 6) are not supported and may cause bugs
If you intend to be able to search for and access text not prefixed by '#' or '###', it must at least be in a section headed by a '###' heading


For example
# How to write a good File
### Headings
Headings must be prefixed by 3 hashtags, no more, no less

### More Markdown

Other markdown can be used such as
- bullet points

> quotes

```js
console.log("Even code")
```

but the program won't pick it up

How the program works

For commands not related to file search, the user must select a file anyway as there is no way for the program to know you don't want to do a file search before you do one.

One file

The user must use the file selector to select one compatible file then press the button and speak an accepted command specifying the word or phrase to find
The program will then search the file given for any h3 headings (with ###) that contain that word or phrase, speak out all options with numbers next to them then prompt the user to select a number that corresponds to a option to proceed All text from the beginning of the selected option's ### to right before the next option's ### will now be searched for the given word or phrase in case the user wants a specific line that doesn't have ### on it
Again, the options are numbered and the user is prompted to select one by number. The program will then read out the option selected

Multiple file

The user must use the file selector to select more than one compatible file then press the button and speak an accepted command specifying the word or phrase to find within the title of the given files (title marked with #)
The program will then search the files given for any h1 headings (with #) that contain that word or phrase, speak out all options with numbers next to them then prompt the user to select a number that corresponds to a option to proceed
The program will then proceed same as above for one file, searching the selected file for ### headings with the given phrase and so on down the chain
